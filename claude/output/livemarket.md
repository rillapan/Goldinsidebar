## Problem Statement

GoldMind AI sudah punya pipeline harga XAUUSD (REST polling → Redis → Socket.IO → Frontend),
tapi tidak ada halaman yang menampilkan data ini secara visual kepada member. Member hanya melihat
harga kecil di header sinyal. Fitur Live Market akan menjadi nilai jual utama membership: member
bisa memantau harga XAUUSD real-time, tren 30 menit terakhir, dan sentiment pasar — semua dalam
satu dedicated page dan mini widget di dashboard.

---

## Demand Evidence

- Member berbayar sudah ada (sistem Xendit + Membership model aktif).
- Harga XAUUSD sudah mengalir ke frontend via `price_update` Socket.IO event — ada demand
  teknikal yang teruji.
- `USD_TO_IDR = 16300` dan `formatIDR()` helper sudah ada di `dashboard/page.tsx` — artinya
  konversi IDR sudah dipikirkan sebagai kebutuhan pengguna sejak awal.

---

## Status Quo

Member saat ini hanya melihat harga live via `price_update` event di dashboard sinyal — ditampilkan
sebagai angka kecil di header card sinyal. Tidak ada visualisasi tren, tidak ada sentiment gauge,
tidak ada dedicated page untuk memantau pasar.

---

## Target User & Narrowest Wedge

**Target:** Member aktif GoldMind AI (ACTIVE status) yang ingin memantau XAUUSD sebelum atau
sesudah membaca sinyal trading.

**Narrowest wedge:** Ticker harga dengan konversi IDR + sparkline 30 menit — dua elemen ini
sudah bisa dibangun dengan data yang ada (Socket.IO + Redis candles), tanpa perlu API baru.

---

## Premises

1. **Harga update setiap 30 detik** — `price_feed.py` menggunakan REST polling (`POLL_INTERVAL=30`).
   WebSocket Twelve Data membutuhkan plan berbayar. Efek "glowing" akan terjadi setiap ~30 detik
   (bukan per detik). Backend `broadcastPrice` tetap jalan setiap 2 detik tapi data Redis tidak
   berubah di antara polling. ✅ Disetujui.

2. **Sparkline dari candle buffer** — `candles:xauusd:1m` sorted set di Redis sudah berisi hingga
   300 entries. Endpoint baru akan membaca 30 entries terakhir (30 menit). ✅ Disetujui.

3. **Live Market hanya untuk member ACTIVE** — gated by `verifyToken` + `checkMembership`
   middleware, konsisten dengan sinyal dan chat routes. ✅ Disetujui.

---

## Approaches Considered

### Approach A: Pure Frontend Buffer
- Tidak ada endpoint baru.
- Client accumulate harga dari Socket.IO ke `useRef` array.
- Sparkline pakai TradingView Lightweight Charts dari ticks yang terakumulasi.
- **Kelemahan:** Cold start — sparkline kosong hingga ~30 detik pertama setelah refresh.
- Effort: S (~2 hari). Risk: Low.

### Approach B: Backend Candles Endpoint (DIPILIH)
- `GET /api/market/candles` — baca `candles:xauusd:1m` dari Redis, return 30 candle terakhir.
- `GET /api/market/status` — health check server-authoritative.
- Frontend fetch candles saat page load → sparkline langsung terisi tanpa cold start.
- Sentiment dihitung dari 30 candle history (bullish/bearish ratio).
- Effort: M (~4 hari). Risk: Low.

### Approach C: WebSocket Upgrade
- Upgrade plan Twelve Data, rewrite `price_feed.py` ke WebSocket.
- True per-second ticks, glowing ticker sungguhan.
- **Kelemahan:** Biaya plan upgrade, rewrite cukup besar, testing lebih panjang.
- Effort: L (~7-10 hari). Risk: Medium.

```typescript
// GET /api/market/candles
// - verifyToken + checkMembership (premium only — data proprietary)
// - Seluruh handler dalam try/catch → jika Redis down (ioredis maxRetriesPerRequest:0 throw immediately):
//     return res.json({ candles: [], count: 0, warning: 'Redis tidak tersedia' })
// - Baca candles:xauusd:1m dari Redis: ZRANGE candles:xauusd:1m -30 -1 (30 entries terakhir)

//   historis, bukan "live feel". 30 adalah sweet spot untuk sparkline.
// - Normalisasi field: sorted set berisi dua format dari price_feed.py:
//     - _price_poll_loop: { price, bid, ask, timestamp, ... }   ← field "price", tidak ada "close"
//     - _ohlcv_refresh_loop: { open, high, low, close, ... }    ← field "close", juga ada "price"=close
//   Gunakan: const value = entry.close || entry.price
//   Alasan pakai || bukan ??: _ohlcv_refresh_loop set "price": float(c.get("close", 0))
//   sehingga jika close = 0 (data invalid), ?? tidak membantu; || skip falsy 0 ke price fallback
// - Jika sorted set kosong (ai-engine belum jalan cukup lama), return candles: [] dengan warning
// Response: { candles: [{price: number, timestamp: string}], count: number }

// GET /api/market/status
// - PUBLIC (tidak perlu auth) — health check tidak membawa data proprietary
// - PENTING: mount SEBELUM router.use(verifyToken, checkMembership) di market.routes.ts:
//     router.get('/status', statusHandler);          // ← public, tidak perlu auth
//     router.use(verifyToken, checkMembership);       // ← gating untuk route lain
//     router.get('/candles', candlesHandler);
// - Baca price:xauusd dari Redis, cek field timestamp
// - Stale threshold: 90 detik (bukan 10 detik — 10 detik false-alarm karena REST poll setiap 30s)
// - Jika tidak ada key → status "stale"
// - Wrap seluruh handler dalam try/catch → jika Redis down, return { status: 'stale', staleSeconds: 999 }
// Response: { status: "live" | "stale", lastUpdate: string, staleSeconds: number }
```

**`goldmind-ai/frontend/src/app/livemarket/page.tsx`**
- `'use client'` directive wajib (menggunakan Socket.IO + TradingView Lightweight Charts)
- Auth-gated: cek session, redirect ke `/login` jika belum login
- Fetch `GET /api/market/candles` saat mount → initial sparkline data
- Listen `price_update` Socket.IO → update ticker + deduplicate + append ke sparkline + recalculate sentiment
- Listen `socket.on('disconnect')` → set status "CONNECTING"
- Listen `socket.on('connect')` → set status "LIVE", clear stale timer

**`goldmind-ai/frontend/src/components/LiveMarket/Sparkline.tsx`**
- `'use client'` + **wajib import via `dynamic(() => import(...), { ssr: false })`** di parent
  karena TradingView Lightweight Charts memanipulasi DOM langsung (crash saat SSR)
- Props: `candles: {price: number, timestamp: string}[]`, `height?: number` (default 200)
- Initial data: map candles ke TradingView format:
  ```typescript

    time: Math.floor(new Date(c.timestamp).getTime() / 1000) as UTCTimestamp,
    value: c.price,
  }));
  ```
- Real-time update via `series.update({time, value})` — jangan append duplikat:
  ```typescript
  // Deduplicate: jika timestamp sama dengan entry terakhir, skip

  lastTime.current = tvTime;
  series.update({ time: tvTime, value: price });
  ```
- Dark/light theme: gunakan `useTheme()` dari `next-themes`, pass warna ke `createChart` options
- MiniWidget reuse: `<Sparkline candles={candles} height={100} />` — satu komponen, height configurable

**`goldmind-ai/frontend/src/components/LiveMarket/SentimentGauge.tsx`**
- SVG needle gauge 180° (half-circle), viewBox `"0 0 200 110"`
- Skala: score 0 = Strong Sell (merah `#CF304A`) → 50 = Neutral (kuning `#F0B90B`) → 100 = Strong Buy (hijau `#03A66D`)
- Degree mapping (pivot di tengah arc):
  ```typescript
  // score 0 → -90° (jarum kiri), score 50 → 0° (tengah), score 100 → +90° (kanan)
  const degrees = (score / 100) * 180 - 90;
  // Needle: transform-origin di cx=100,cy=100; rotate(degrees)
  ```

**`goldmind-ai/frontend/src/components/LiveMarket/SystemStatus.tsx`**
- Props: `status: 'live' | 'stale' | 'connecting'`
- Dot: `●` hijau = "LIVE", kuning = "STALE", abu animasi = "CONNECTING"
- Di parent (`page.tsx`): stale timer menggunakan threshold **60 detik** (bukan 10 detik)
  karena REST poll 30s + network jitter bisa membuat 10s false-alarm
  ```typescript
  useEffect(() => {
    const timer = setTimeout(() => setStatus('stale'), 60_000);
    return () => clearTimeout(timer);
  }, [lastPriceUpdate]); // reset setiap price_update masuk
  ```
- Wiring Socket.IO:
  ```typescript
  socket.on('disconnect', () => setStatus('connecting'));
  socket.on('connect',    () => { setStatus('live'); resetStaleTimer(); });
  socket.on('price_update', () => { setStatus('live'); resetStaleTimer(); });
  ```

### Sentiment Gauge — Detail Kalkulasi

```typescript
function calculateSentiment(candles: {price: number}[]): number {
  if (candles.length < 2) return 50; // neutral default
  let bullish = 0;
  for (let i = 1; i < candles.length; i++) {
    if (candles[i].price > candles[i-1].price) bullish++;
  }
  return Math.round((bullish / (candles.length - 1)) * 100);
}
// 0-30 = Strong Sell, 30-45 = Sell, 45-55 = Neutral, 55-70 = Buy, 70-100 = Strong Buy
// Timeframe: 30 candle (30 menit) — lebih bermakna dari 4 data points di 120 detik
```

### Health Check — Detail

Backend `GET /api/market/status` (PUBLIC, tidak perlu auth):
```typescript
const raw = await redis.get('price:xauusd');
if (!raw) return res.json({ status: 'stale', staleSeconds: 999 });
const data = JSON.parse(raw);
const staleSeconds = (Date.now() - new Date(data.timestamp).getTime()) / 1000;
return res.json({
  status: staleSeconds > 90 ? 'stale' : 'live',  // 90s threshold (bukan 10s)
  lastUpdate: data.timestamp,
  staleSeconds: Math.round(staleSeconds),
});
```

### Edge Case: Candle Data Kosong

Jika `candles:xauusd:1m` di Redis kosong (ai-engine belum jalan ≥30 menit):
- Backend return `{ candles: [], count: 0 }`
- Frontend: tampilkan placeholder "Menunggu data..." di area sparkline
- Ticker masih bisa menampilkan harga realtime dari `price_update` Socket.IO
- Sparkline mulai mengisi sendiri setelah data terkumpul (client-side accumulation sebagai fallback)

## The Assignment

Sebelum mulai coding: pastikan `candles:xauusd:1m` Redis punya data yang cukup dengan menjalankan
ai-engine minimal 30 menit dan cek via `redis-cli ZCARD candles:xauusd:1m` — harusnya ada minimal
6 entries (30 menit / 5 menit OHLCV refresh). Jika nol, ada masalah di `price_feed.py` yang harus
diselesaikan dulu sebelum sparkline bisa ditest.

**Catatan nama key:** `candles:xauusd:1m` adalah nama historis dari desain awal. Dalam praktiknya
isi key ini adalah OHLCV candle yang di-refresh setiap 5 menit oleh `_ohlcv_refresh_loop`, BUKAN
1-menit candle yang diupdate setiap menit. Ini tidak mempengaruhi implementasi — API endpoint tetap
baca `ZRANGE candles:xauusd:1m -30 -1` — tapi perlu dipahami agar tidak bingung saat debugging.
